## Why

当前 Agent 详情页面的"最近任务"列表展示方式不够直观，用户需要多次点击才能执行任务和查看结果。聊天窗口式的命令交互界面可以提供更自然、高效的操作体验，让运维人员能够在一个页面内快速输入命令并实时查看执行结果，提升运维效率。

## What Changes

- **Agent 详情页重新设计**
  - 将"最近任务"列表替换为聊天窗口式命令终端界面
  - 左侧区域：显示命令执行历史和结果（机器响应）
  - 右侧区域：命令输入框和执行按钮

- **任务执行功能整合**
  - 将任务管理页面的"执行任务"功能迁移到 Agent 详情页
  - 支持快速执行 Shell 命令和内置命令
  - 提供命令历史记录（上下箭头浏览）

- **实时输出展示**
  - 支持任务执行过程中的实时输出流式显示
  - 区分命令输入（右侧/用户）和执行结果（左侧/机器）的视觉样式
  - 支持执行状态指示（运行中/成功/失败/超时）

- **命令快捷操作**
  - 提供常用命令快捷按钮（如：`ls -la`, `top`, `df -h`）
  - 支持命令自动补全（基于历史命令）
  - 支持命令输入框多行输入（Shift+Enter 换行，Enter 执行）

## Capabilities

### New Capabilities
- `agent-terminal-chat`: Agent 终端聊天界面，提供聊天式命令执行和结果展示功能

### Modified Capabilities
- `dashboard-frontend`: 添加 Agent 终端聊天界面到 Agent 详情页
- `quick-task-exec`: 整合快速任务执行流程到终端聊天界面

## Impact

**前端代码**
- 新增或修改组件：`AgentTerminal.tsx`（聊天容器）、`CommandInput.tsx`（命令输入）、`CommandHistory.tsx`（历史记录）
- 修改页面：`agents/[id]/index.tsx`（Agent 详情页布局调整）
- 修改样式：终端聊天界面的 CSS（深色主题等）

**后端代码**
- 复用现有 Task API，无需新增接口
- 可能需要增强 WebSocket 的实时输出推送功能（已在 `task-execution` 规范中定义）

**用户体验**
- 提升操作效率：减少页面跳转，一个页面完成命令执行
- 降低学习成本：聊天式界面更直观
- 增强实时反馈：流式输出让用户即时看到执行进度
