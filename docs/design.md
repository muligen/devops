# Windows Agent 设计文档

## 1. 项目概述

Windows Agent 是一个运行在 Windows 机器上的客户端程序，用于远程管理和控制。支持接收服务器命令、执行任务、上报状态，并具备自更新能力。

## 2. 技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| 编程语言 | C++ | 便于调用 Windows API，高性能 |
| 依赖管理 | Conan | C++ 主流包管理器 |
| 构建系统 | CMake + MSVC | Windows 平台最佳实践 |
| 通信协议 | WebSocket over TLS | 双向实时通信，穿透性好 |
| 消息格式 | JSON | 可读性好，调试方便 |
| 配置格式 | YAML | 人类可读，配置友好 |

## 4. 功能模块

### 4.1 次任务（工作进程1）

| 功能 | 频率 | 说明 |
|------|------|------|
| 心跳上报 | 每秒 | 向服务器发送心跳，维持在线状态 |
| 数据上报 | 每分钟 | 上报内存、硬盘容量等机器数据 |

**上报数据结构：**

```json
{
  "type": "metrics",
  "agent_id": "agent-001",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "cpu_usage": 45.2,
    "memory": {
      "total": 16384,
      "used": 8192,
      "percent": 50.0
    },
    "disk": {
      "total": 512000,
      "used": 256000,
      "percent": 50.0
    },
    "uptime": 86400
  }
}
```

### 4.2 主任务（工作进程2）

| 命令类型 | 说明 | 参数示例 |
|----------|------|----------|
| `init_machine` | 按配置初始化机器 | `{"config_url": "..."}` |
| `clean_disk` | 按规则清理磁盘 | `{"rules": ["temp", "cache", "logs"]}` |
| `exec_shell` | 执行 Shell 命令 | `{"command": "ipconfig", "timeout": 30}` |

## 5. 通信设计

### 5.1 协议选型

```
┌─────────────────────────────────────────────────────────────────┐
│                 WebSocket over TLS (WSS)                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  优势：                                                          │
│  ✓ 双向实时通信 - 服务器可主动推送命令                           │
│  ✓ 穿透性好 - 走 HTTPS 443 端口，防火墙友好                      │
│  ✓ 断线重连 - 网络恢复后自动重连                                 │
│  ✓ 成熟稳定 - 各语言库支持完善                                   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 消息类型

| 方向 | 类型 | 用途 | 示例 |
|------|------|------|------|
| Agent → Server | `heartbeat` | 心跳保活 | `{"type": "heartbeat", "agent_id": "..."}` |
| Agent → Server | `metrics` | 机器数据上报 | `{"type": "metrics", "data": {...}}` |
| Agent → Server | `command_result` | 命令执行结果 | `{"type": "command_result", "id": "...", "status": "success"}` |
| Agent → Server | `output_chunk` | 实时输出分片 | `{"type": "output_chunk", "id": "...", "output": "..."}` |
| Server → Agent | `command` | 下发命令 | `{"type": "command", "id": "...", "cmd": "init_machine", ...}` |
| Server → Agent | `update` | 更新通知 | `{"type": "update", "version": "2.0.0"}` |

### 5.3 消息格式示例

**命令下发：**
```json
{
  "type": "command",
  "id": "cmd-12345",
  "cmd": "exec_shell",
  "params": {
    "command": "ipconfig /all",
    "timeout": 30
  },
  "timestamp": "2024-01-15T10:30:00Z"
}
```

**命令结果：**
```json
{
  "type": "command_result",
  "id": "cmd-12345",
  "status": "success",
  "exit_code": 0,
  "output": "Windows IP Configuration...",
  "duration": 1.5,
  "timestamp": "2024-01-15T10:30:02Z"
}
```

## 6. 安全设计

### 6.1 多层安全架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         安全层次                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Layer 1: 传输层                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  TLS 1.3 加密通信                                         │   │
│  │  • 防止中间人攻击                                         │   │
│  │  • 防止流量窃听                                           │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
│  Layer 2: 身份认证                                              │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  AgentID + Token 机制                                     │   │
│  │  • 注册时获取唯一 ID 和 Secret Token                      │   │
│  │  • 连接时 Challenge-Response 验证                         │   │
│  │  • 可选：mTLS 双向证书认证                                │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │

│  │  • 禁止任意命令注入                                       │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 7. 部署设计

### 7.1 运行模式

**Windows Service（服务）**

- 开机自启，无需用户登录
- 后台运行，用户无感知
- 异常崩溃后自动恢复
- 通过 `sc.exe` 或 `services.msc` 管理

### 7.2 目录结构

```
C:\Program Files\AgentTeams\
├── bin\
│   ├── agent.exe              # 主进程
│   ├── worker_heartbeat.exe   # 工作进程1
│   └── worker_task.exe        # 工作进程2
├── config\
│   └── agent.yaml             # 配置文件
├── logs\
│   ├── agent.log              # 主进程日志
│   ├── heartbeat.log          # 心跳日志
│   └── task.log               # 任务日志
└── temp\                      # 临时文件（更新下载等）
```

### 7.3 配置文件

```yaml
# agent.yaml
server:
  url: wss://server.example.com/agent
  reconnect_interval: 5s
  heartbeat_timeout: 30s

auth:
  agent_id: "agent-001"
  token: "your-secret-token"

task:
  max_concurrent: 8        # 最大并发任务数
  default_timeout: 5m      # 默认超时
  queue_size: 100          # 命令队列大小

heartbeat:
  interval: 1s             # 心跳间隔

metrics:
  interval: 1m             # 数据上报间隔

logging:
  level: info
  max_size: 100MB
  max_files: 10

update:
  check_interval: 1h       # 更新检查间隔
  idle_required: true      # 需要空闲时才更新
```

## 8. 错误处理与恢复

### 8.1 异常处理

| 场景 | 处理策略 |
|------|----------|
| 网络断开 | 指数退避重连（5s → 10s → 30s → 60s 上限） |
| 服务器无响应 | 心跳超时后重连 |
| 工作进程崩溃 | 主进程检测并自动重启 |
| 命令执行超时 | 强制终止子进程，返回超时状态 |
| 更新失败 | 回滚到旧版本，上报错误 |

### 8.2 日志记录

```
日志级别：
  - ERROR: 错误信息，需要关注
  - WARN:  警告信息，可能有问题
  - INFO:  常规信息，连接、命令等
  - DEBUG: 调试信息，详细流程

日志轮转：
  - 单文件最大 100MB
  - 保留最近 10 个文件
  - 自动清理过期日志
```

## 11. Server 端设计

### 11.1 设计原则

| 原则 | 说明 |
|------|------|
| 模块化单体 | 单一仓库，模块边界清晰，未来可拆分为微服务 |
| 事件驱动解耦 | 核心流程通过消息队列解耦，便于扩展、重试、审计 |
| 领域驱动设计 | 按业务领域组织代码，而非技术层 |
| API First | 先定义 API 契约，再实现，便于前后端并行 |

### 11.8 敏捷开发计划

| Sprint | 交付内容 |
|--------|----------|
| Sprint 1 | Agent 模块 + 基础架构（WebSocket 连接、认证） |
| Sprint 2 | Task 模块 + 命令执行（任务下发、结果回传） |
| Sprint 3 | Monitor 模块 + 仪表盘（心跳、数据展示） |
| Sprint 4 | Update 模块 + 自更新（版本管理、分发） |
| Sprint 5 | User 模块 + 权限控制（用户管理、审计） |
