# 部署验证指南 - Agent 终端聊天界面

本文档提供 Agent 终端聊天界面功能部署后的验证步骤。

## 部署前检查清单

### 代码质量
- [ ] 所有 lint 错误已修复：`npm run lint`
- [ ] 类型检查通过：`npm run build`
- [ ] 单元测试通过：`npm run test`
- [ ] 浏览器无控制台错误

### 依赖检查
- [ ] 后端 WebSocket 服务正常运行
- [ ] 任务 API (`taskApi.createTask`) 可访问
- [ ] localStorage 支持正常（所有目标浏览器）

## 部署后功能验证

### 1. 基础功能验证

#### 访问终端界面
**步骤**:
1. 登录系统
2. 进入「Agent」页面
3. 点击任意 Agent
4. 选择「终端」标签页

**预期结果**:
- 终端界面正常显示
- 输入框可见且可用
- 状态栏显示 Agent 在线/离线状态

#### 命令执行
**步骤**:
1. 在输入框输入 `ls -la`
2. 按 Enter 键

**预期结果**:
- 命令气泡（蓝色）出现在消息列表
- 执行结果气泡（灰色/绿色）出现
- 状态从「执行中...」变为完成状态
- 显示退出码和执行时长

### 2. 命令历史验证

**步骤**:
1. 执行多个命令：`pwd`, `ls`, `whoami`
2. 在输入框按 ↑ 箭头键
3. 继续按 ↑ 浏览历史
4. 按 ↓ 返回

**预期结果**:
- ↑ 键显示历史命令（从旧到新）
- ↓ 键返回空输入或较新的命令
- 刷新页面后历史记录仍然保留

### 3. 快捷命令验证

**步骤**:
1. 点击左侧快捷命令按钮（如「列出文件」）
2. 观察命令自动执行

**预期结果**:
- 命令自动填充到输入框并执行
- 结果正确显示

### 4. 多行命令验证

**步骤**:
1. 输入 `echo "line"`
2. 按 Shift+Enter 换行
3. 输入 `echo "line2"`
4. 按 Enter 执行

**预期结果**:
- 命令正确包含两行
- 执行结果显示两行输出

### 5. 消息操作验证

**复制输出**
**步骤**:
1. 点击结果消息右上角「复制」按钮
2. 在任意文本框粘贴

**预期结果**:
- 输出内容被正确复制到剪贴板

**清空终端**
**步骤**:
1. 点击顶部「清空」按钮

**预期结果**:
- 所有消息被清除
- 命令历史保留

### 6. Agent 切换验证

**步骤**:
1. 在 Agent A 终端执行命令
2. 返回 Agent 列表
3. 打开 Agent B 终端
4. 再返回 Agent A 终端

**预期结果**:
- Agent B 显示独立的消息历史
- Agent A 的消息历史正确保留

### 7. 离线状态验证

**步骤**:
1. 选择离线的 Agent
2. 尝试输入命令

**预期结果**:
- 输入框被禁用
- 状态栏显示「离线」

### 8. 错误处理验证

**步骤**:
1. 执行会失败的命令（如 `invalidcommand`）

**预期结果**:
- 显示红色错误消息气泡
- 错误信息清晰可读

- [ ] 长输出展开无明显卡顿
